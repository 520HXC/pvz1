# Tower Defense Designer

Use this skill when tuning tower-defense gameplay identity.

## Design Priorities

- Readability first: each unit should have a clear role and counterplay.
- Distinct pressure types: rush, armor check, lane split, utility disruption.
- Fair pacing: quiet windows between spikes.
- Progression identity: early, mid, late bands should play differently.

## Unit Role Matrix

- Economy units: setup value, low direct DPS.
- Core DPS: consistent lane defense.
- Burst/control: panic tools, stun/slow/displacement.
- Armor/support: defensive sustain and protection.
- Special threats: force specific answers and adaptation.

## Level Curation Workflow

1. Define band identity (tutorial/night/pool/fog/roof/mixed).
2. Curate zombie composition by pressure theme, not random scaling.
3. Curate card pool by learning objective and available counters.
4. Tune start resources and spawn pacing for fairness.
5. Add danger labels and threat previews for player planning.

## Fairness Rules

- Never require unavailable counters.
- Avoid stacking multiple high-complexity threats too early.
- Keep start economy recoverable after one mistake.
- Keep final boss clearly harder but still solvable.

## Output Pattern

- Mention exact parameters changed.
- Explain why each change improves fairness or clarity.
- Highlight any remaining high-risk combos.
