# UI Design Finder

Use this skill when the user provides visual references and wants a cleaner, more intentional UI rebuild.

## Main Objective

- Convert inspiration images into actionable layout specs.
- Keep composition and hierarchy, not pixel-copy assets.
- Produce reusable design language across scenes.

## Process

1. Read the target scene intent:
- primary action
- secondary actions
- supporting info
- navigation path

2. Define structural regions first:
- top/header
- main action block
- side info block
- bottom utility/action area

3. Set style vocabulary:
- panel types
- button families
- title/subtitle scale
- spacing grid and margins

4. Build component map:
- exact helper draw functions needed
- where each helper is reused

5. Validate usability:
- no overlaps
- no clipped text
- primary button visibility
- back navigation always reachable

## Output Expectations

- List old regions to discard.
- List new scene regions with rectangle responsibilities.
- List helper components and where used.
- Explain how final composition tracks the reference mood.

## Constraints

- Do not copy copyrighted textures/art directly.
- Do preserve gameplay behavior unless asked otherwise.
